# Current Game State

Last reviewed: 2026-07-05

This document maps the current implemented game progression from the static game data and runtime unlock rules. It is based on the active data in `src/data/**`, the Explore filtering logic in `src/components/game/features/explore/hooks/useExploreData.ts`, and the requirement resolver in `src/utilities/requirements/index.ts`.

## Runtime Notes

Recent maintenance touched several stateful game systems:

- Pokedex caught-form research notes show authored TM unlocks from `src/data/pokemon-research-level-rewards.json` by research rank, with unreached unlocks masked as `???`.
- Pokemon detail runtime controls now allow clearing assigned battle moves from the details drawer, which lets players recover Pokemon whose saved move loadout no longer passes current TM/form/level rules. Owned Pokemon ability cards show the canonical ability description plus generated Partner Effect rows for any Capture/Field Observation bonuses. Evolution now preserves the source Pokemon's natural ability slot on the evolved form, so hidden abilities remain hidden abilities and normal slot abilities map to the evolved form's matching slot when available.
- Wild catch rewards now include one guaranteed base-type gem per qualifying wild battle reward pass, and temporary drop tuning now suppresses all tier-2/3 material or gem rewards in drop-table-style generation (tier-2/3 IDs remain authored but are not emitted by these paths). Catch-focused dowsing random events were also converted from Lure Cache events to Gem Cache events, now yielding seven gems.
- Colored berry candies now increase friendship by 5.
- Reward result overlays queue task exit modals in order: the task just completed shows its own `exitModal` first, then any exit modals from hidden tasks completed as rewards, such as recipe manual unlock notices. Activity rewards that grant `task_complete` now also cascade that completed task's own rewards into the same reward result.
- User-owned ledgers live in normalized Payload collections. `getGameUserData` reads `user-inventory-items`, `user-pokedex-entries`, `user-abilitydex-entries`, `user-task-progress`, `user-activity-stats`, `user-tcg-cards`, and `user-shop-purchases` without runtime legacy JSON fallback. Runtime write paths use the normalized user-state helpers directly, the old `users` after-change mirror hook and legacy migration helpers are removed, and old ledger columns are dropped from `users`. Client code consumes top-level `RequirementData` arrays through `gameData` instead of compatibility-shaped `user` ledgers. `/api/game/sync` returns a slim `user` object plus those arrays, and its default scope excludes full Pokemon collection hydration; Explore still requests its scoped slim Pokemon fields.
- Battles render through a visual state queue in `src/utilities/battle/engine/useBattleManager.ts`. KO replacements must not be treated as normal swaps before the KO sequence, or fainting Pokemon disappear before the faint animation can play. Forced post-KO replacements are switch-in-only visual events and must not replay the previous combat log. While a player is choosing a forced replacement, the fainted active slot stays visually empty until the replacement switch-in starts. Battle damage uses a shared 50 BP baseline for normal stance attacks and 1x battle moves before stance, stats, type, weather, held-item, secondary-status reduction, and other modifiers. Successful damaging attacks carry their resolved type through battle-log metadata into target impact animations only when that side wins the stance matchup, so Fire, Water, Electric, and the other Pokemon types can render matching hit effects without changing damage timing. Battle starts now keep the resolved sub-region weather snapshot for the whole fight; the battle engine applies Pokemon-style weather damage, defense, accuracy, freeze-immunity, primal-weather blocking, and end-turn chip effects from that snapshot. Secondary battle statuses are log-only effects attached to Pokemon, sides, or the field; they can trigger before moves, after normal end-turn damage/weather, or on switch-in; optionally delay before first effect; and support damage and absorb with optional Pokemon-type filters, healing, regular status application, status immunity, wake-status effects, stat stages, outgoing stance/type damage modifiers, incoming stance/type damage reduction or damage-taken modifiers, passive accuracy bypass, type-immunity bypass effects that make matching 0x damage resolve as neutral 1x damage, infatuation, voluntary switch prevention, and move blocking including healing-move prevention. Reapplying the same secondary status ID to the same Pokemon, side, or field refreshes/replaces it instead of stacking duplicates. Pokemon-attached secondary statuses, including Yawn-style delayed sleep, Wish-style delayed healing, Uproar-style wake/status prevention, Attract-style gender-gated infatuation, trapping effects such as Block/Mean Look/Wrap, and Foresight/Odor Sleuth-style Identified exposure, clear when that Pokemon switches out or when a move authors `secondaryStatusCure`; cures can now target Pokemon statuses, side statuses, or both, and can optionally restrict clearing to specific secondary-status IDs. Foresight and Odor Sleuth intentionally ignore evasion reset because this game has no evasion stat. Rapid Spin clears the user's Pokemon and side secondary statuses, while Tidy Up clears authored hazards plus Substitute/Shed Tail decoys from both active sides without removing unrelated effects. Infatuation requires opposite non-genderless genders and gives a 50% before-move immobilization roll. Switch-prevention secondary statuses block voluntary player, PVP, and trainer AI switches, but forced KO replacements and move-forced switches still resolve. Source-linked trapping statuses also clear when the Pokemon that caused them leaves battle. Move contests can now use Pokemon friendship as a metric for moves such as Pika Papow. Charged assigned moves spend generic non-telegraphed charge turns before release, with Vanished charge moves failing before spending the move if Vanished cannot be applied, recharge moves lock the player into a post-use Recharge action that skips attacking while the enemy acts, and continuous assigned moves lock the player into the same move until their rolled duration ends or the move misses, fails, or is interrupted; Echoed Voice and Rollout can scale damage on successful repeats, while rampage-style moves can apply end-of-sequence self-confusion. Enemy-authored PVE moves now use the same effective-Speed contests, stance-disable effects, delayed damage queues, charged turns, recharge turns, continuous locks, and self-pivot switches; player-side battle rewards remain player-only. Runtime move effects can transfer, remove, steal, swap, recycle, and consume held berries in battle, while held plates, memories, and drives are dynamic type sources for Judgment, Multi-Attack, and Techno Blast; assigning or removing those held items also syncs Arceus, Silvally, and Genesect between their matching stored forms. Runtime move effects also cover next-accuracy bypass, conditional damage including no-held-item, target Pokemon type, target-Dynamax, and target-status checks, target-status and consumed-held-item use gates, post-damage status cures, main-status transfers such as Psycho Shift, weather-aware healing, partial or full stat-stage copy/swap effects, move locks, Transform-style battle copying, and target-primary/user-primary type changes. Camouflage uses a runtime type-change effect to randomly set the user's active battle type, Conversion sets the user's active battle type to its first known battle move's type, Conversion 2 sets the user to a type that resists or ignores the opponent's last successful authored move or basic stance attack type, and Weather Ball changes type from active battle weather while doubling from 50 BP to 100 BP in non-clear weather. Disable now disables one random base stance for two turns. Rage is weak Normal damage that doubles only after the user recently took damage, rather than granting Attack immediately on use. Pay Day uses a player-side battle reward effect to add 30 Pokédollars to the win rewards. Weather Control is a Gen 3-style Pokemon Power unlocked by `weather-orb` and per-weather `weather-core-*` key items; it overwrites only the active battle weather until another battle weather override or battle end. The battle UI now has mobile/PWA recovery watchdogs that snap to the latest confirmed server state if Safari strands a visual queue or a client action remains on "Resolving action" too long. Client-side battle diagnostics keep the latest compact visual-queue breadcrumbs in `localStorage` under `pokemon-app:battle-debug-log` for crash investigation.
- Player-owned Pokemon documents keep a persistent `battleKOs` counter for opposing Pokemon they knocked out in PvE or PvP battles. KO credits are recorded on the active battle state when a faint is caused by recorded damage, then flushed onto the Pokemon document when the battle reaches a terminal result or surrender.
- Regular PvE wild/trainer enemies roll missing IVs and EVs at battle start instead of using fixed defaults. Wild Pokemon roll each missing IV once, have 0 EVs through level 40, and roll missing EVs from 1-100 above level 40. Trainer Pokemon roll each missing IV once below level 30, twice and keep the higher roll from level 30, and three times and keep the highest roll from level 50. Trainer missing EVs roll per stat from 1-70 below level 20, 1-150 from level 20, 50-252 from level 50, and 100-252 from level 70. Manual enemy IV/EV stat overrides still take precedence.
- Battle HP playback separates pre-combat item healing, combat damage, and explicit `[icon:damage:n]`/`[icon:heal:n]` secondary or status ticks from the same server turn log, so complex turns animate toward the server's final HP instead of visibly snapping after the queue sync.
- Battle actions carry a client action id through turn, move, item, swap, and battle-power server actions. Completed results are cached briefly and a per-user battle action lock prevents retry or double-tap requests from resolving the same turn twice; ongoing item, swap, and PVP turn results skip route revalidation because the client receives the fresh battle state directly. PVP waiting states poll for the resolved shared turn after the local move has been queued, so the UI can advance without a manual refresh when the opponent's action lands.
- Protect, Detect, and Quick Guard are one-turn 100% damage-reduction secondary-status moves that take hold without stance-loss gating. Self/side damage-reduction secondary statuses apply before incoming damage is reduced so same-turn protection blocks retaliation instead of appearing after the hit. Specialty guard moves now spread 75% damage reduction across the stance triangle: Baneful Bunker and King's Shield cover Power, Spiky Shield and Obstruct cover Speed, and Wide Guard plus Burning Bulwark cover Tech.
- Enemy authored move effects follow the same success semantics as player moves: non-failed enemy moves apply post-damage effects even if the hit KOs the player or deals 0 final damage, effect-only moves count as successful move history, and enemy recharge/continuous locks are based on successful execution rather than final damage dealt. Enemy AI now paces low-value effect moves against stance attacks, penalizes immediate/repeated move use by profile, avoids refreshing active secondary statuses unless they are expiring, reserves Protect-style damage reduction for meaningful incoming damage, does not intentionally select enemy-targeted sleep application while the target is already asleep, and only selects Mimic/Copycat-style called moves when a valid opponent move exists to copy.
- Move-caused held item transfer, theft, swapping, throwing, removal, and berry destruction now stay battle-local; only normal held item consumption from item data, such as berries triggering or breakable held items breaking, can persist after battle.
- Pokemon now store held-item charge on `pokemon.itemCharge`. Removing or replacing a held item resets charge to 0. Inferior Fire/Water/Leaf/Thunder/Shiny/Ice/Dark/Light Stones are held charge items unlocked through the hidden `Recipe: Elemental Stones` task; they craft only on Perfect quality from 10 matching gems, 100 matching T1 material, 1 Neutral Stone, and 1000 Crystals. Fire/Water/Leaf/Thunder/Shiny/Ice gain 5% charge when hit by the matching attack type, with Shiny using Fairy, and at 100% they remove themselves and grant the matching evolution stone. Dark and Light gain 2% charge from any damaging attack only during their active time windows: Light from 06:00-17:59 and Dark from 18:00-05:59. Light grants Dawn Stone if it completes from 06:00-06:59, otherwise Sun Stone; Dark grants Dusk Stone if it completes from 18:00-18:59, otherwise Moon Stone.
- Pokemon sprites resolved through the shared `getPokemonImageUrl` helper, including research mini-games, use bundled local assets from `public/sprites/pokemon` generated by `bun run sync:pokemon-sprites` from the PokeAPI sprites repo. The generated manifest in `src/data/pokemon-sprite-manifest.json` covers app-known forms and falls back to base Unown (`201`) for any missing Pokemon sprite. The unified `bun run generate:game-data` pipeline now runs the Pokemon sprite sync after Pokemon modules are generated, unless `--skip-sprites` is passed, and regenerates the item sprite manifest after TCG and mega-stone item generation, unless `--skip-item-sprites` is passed. All bundled Pokemon sprites are emitted as AVIF during sync; HOME normal/shiny sprites are additionally resized to transparent 200x200 files, while Gen V Black/White battle sprites keep their source dimensions. Female HOME and Gen V sprite variants are bundled and used for owned or battle Pokemon with `gender: female` when a source asset exists. Pokemon rarity is a mutually exclusive rendering field; its CSS-only registry includes normal/shiny references plus Shadow, Radiant, Silver, Gold, Emerald, Ruby, Sapphire, Crystal, Retro, Galactic, Levin, Inferno, Prism, Rainbow, Celestial, Black, White, Void, and Glitch. Glitch preserves the original sprite palette and uses internal animated RGB scan fragments; Black and White are clean high-contrast monochrome treatments. Runtime item sprites now resolve through `src/data/item-sprite-manifest.json` to manually maintained bundled files under `public/sprites/items`; `bun run sync:item-sprites` only scans existing local item sprites and regenerates `docs/game-data/missing-item-sprites.md` for authored items that still need custom art. The item sprite manifest expects evolution items under `public/sprites/items/evolution/`, generated mega stones under `public/sprites/items/mega/`, TMs under `public/sprites/items/materials/`, and generated TCG pack/binder sprites under `public/sprites/items/tcg/`. TCG set logos and symbols are cached in `source_data/tcg/` for `bun run sync:tcg-sprites` pack/binder item sprite composition but are not emitted as public runtime assets. Booster pack use now skips the old pack-opening animation and opens directly into the card drawer reveal. Battle type badges and Pokedex type badges now use fetched PokeAPI type artwork bundled as AVIF under `public/sprites/types`. The PWA service worker keeps a runtime cache for bundled Pokemon sprites and direct sprite proxy responses to reduce repeat fetches and iOS Safari decode pressure during animations. Pokemon Snap additionally preloads and decodes every HOME and Gen V sprite in its appearance pool before starting or resuming visible rounds in the timed session. Field Observation now uses local shiny HOME, local shiny Gen V front, then Unown fallback rather than inverted normal sprites, and renders observation sprites without per-image drop shadows. Pokemon detail modal backgrounds normalize old stored image/background paths to `/backgrounds/*.avif` so pre-rework caught Pokemon do not request removed PNG/JPG background assets.
- Rarity treatments now also include Ancient, a weathered fossil-stone texture; Toxic, a lime-violet poison-bubble treatment with a strong silhouette-bound purple aura; and Pixelated, a crisp chunky pixel-grid treatment. Glitch uses continuous internal RGB scan-band displacement rather than resting between pulses.
- Pokedex detail drawers use existing `/backgrounds/*.avif` habitat art for seen Pokemon across cave, forest, grassland, mountain, rare, rough terrain, sea, urban, and water's edge habitats, with a generic Pokedex background for unseen or unmapped entries.
- The Trainer page now owns the Collection tab, which summarizes Pokedex, Carddex, MoveDex, and AbilityDex progress and links to their detail pages. The old `/game/collection` route and main navigation entry have been removed. Collection category cards use item sprites: Poke Ball for Pokedex, Base Set booster pack for Carddex, Normal TM for MoveDex, and Ability Patch for AbilityDex. MoveDex shows every authored TM/HM move as a compact full-width list row with a move-type icon, known move name or unknown marker, type/stance metadata, and a level chip; unknown/unowned TMs remain blanked, the page filters by move type from a sticky bottom control, and known moves open into their move data and compatible-Pokemon list using the same Pokedex seen/caught visibility rules as the TM inventory modal. MoveDex also has a recovery action that grants any missing unique TM items the player already qualified for through Pokemon Research level rewards before those rewards existed. AbilityDex shows every authored ability as a compact full-width list row with a learner Pokemon sprite when visible, known ability name or unknown marker, and type metadata; unregistered abilities use a greyscale seen compatible Pokemon if possible or the unknown icon otherwise, registered abilities use a coloured compatible Pokemon sprite, and registered entries open into their canonical description, generated Partner Effect rows for Capture/Field Observation bonuses, and compatible form list.

Canonical main-game ability definitions are generated from the local PokeAPI CSV cache into `src/data/abilities/canonical-generated.ts` and cover all generated Pokemon forms. Hidden ability slots roll first at 5%, while normal ability slots are guaranteed fallbacks. The active registry is canonical-only. Every canonical ability resolves to at least one battle effect; exact engine-supported abilities use precise effects, while unsupported cartridge mechanics such as doubles redirection, true contact-only filtering, full cartridge grounded overrides, and move-flag-specific boosts are approximated with thematic stat or damage effects. `docs/game-data/ability-implementation-audit.md` tracks known bad approximations and remaining ability engine gaps. These secondary effects do not alter shiny odds, replace encounters, change encounter levels, or fail encounters. Battle abilities apply only from the Pokemon currently fighting, while Capture/Field Observation secondary effects apply from the active companion ability stored on the encounter or research state. ## Progression Model

Explore combines static entries from:

- `src/data/tasks`
- `src/data/locations`
- `src/data/battles`
- `src/data/games`
- `src/data/shops`
- `src/data/voyages`
- `src/data/expeditions`

Entries become visible when all `requirements` pass. `criteria` are the conditions for completion, entry, or cost. Completed non-repeatable tasks disappear from the Explore list. Secret tasks stay hidden until completed. `hide` removes an entry once a specific task is complete, and `overrides` lets a newly unlocked entry replace an older one.

Tasks marked with `chat: true` are dialogue entries. Explore renders them in a Conversations section, uses a chat bubble type icon, and does not decorate them as repeatable tasks even when their data is repeatable.

Player state that drives progression is mainly:

- Completed tasks: `user-task-progress`
- Battle, location, research, and expedition results: `user-activity-stats`
- Inventory: `user-inventory-items`
- Currencies
- Held item assignment on owned Pokemon
- Pokédex seen/caught and research levels: `user-pokedex-entries`
- Skills: `catching`, `battling`, `researching`, `artisan`, `ranked-battling`. The core player-facing skills are Explorer, Trainer, Researcher, and Artisan.
- Partner and battle team assignments. A Pokemon can occupy only one active roster role at a time: boxed, partner, or battle team.
- Voyage and expedition completion stats

## Main Game Flow

### 1. Pallet Town Tutorial

The game begins with `tutorial-1` / "A Fresh Start", which grants a `starter-ball`. The player spends that ball on one starter selection task. Starter selection is implemented as multiple parallel tasks gated by `item_owned: starter-ball` with `consume: true`, and all gifted starters join at level 5.

After the starter is chosen, `pallet-town-orientation` becomes the mandatory guided tutorial expedition. It cannot be abandoned, failed steps stay retryable without consuming lives or deleting the run, and it keeps Pallet Town closed until the player completes a compact sequence:

1. Oak's lab briefing.
2. `pallet-orientation-cry` cry-recognition research, framed as identifying the Pokemon chewing through Oak's cable cabinet.
3. Battle-team briefing.
4. `pallet-orientation-lab-rattata` first battle.
5. Field kit handoff.
6. `pallet-orientation-field-observation` Route 1 Study.
7. `pallet-orientation-route-1` first Route 1 catch.
8. `pallet-orientation-rival-selection`, where Oak asks which trainer the player expects to keep meeting on the road after the basic field checks.
9. Final Oak wrap-up, where Oak directs the player to speak with people in the lab and explore Pallet Town before leaving.

Claiming the expedition reward marks hidden `tutorial-16` / "Adventure Awaits!" complete and grants the `kanto-badge-case` key item. The guided orientation steps do not grant direct skill XP or Escape Rope rewards; `pallet-artisan-soft-touch-briefing` explains that Route 1 Soft Fluff can be used at the Artisan bench, then the required `pallet-artisan-basics` glove-patching task must be completed before the post-orientation Pokedex check handles the first Escape Rope handoff. Normal Route 1 Catch, Battle, and Study entries are locked behind `tutorial-16`, so open Pallet content does not appear before the guided expedition is finished.
Route 1 Catch, Battle, and Study include a gated Alolan Rattata entry during Kanto nighttime from 18:00-06:00 once the player reaches Researcher level 5 and Explorer level 5. Catch, Field Observation Pokemon pools, and wild battle enemy pools support per-Pokemon requirements for these conditional additions.
Normal pre-Brock battle content now ramps instead of using a flat cap: Route 1, Grumpy Man, and Route 22 cap player Pokemon at level 7; the Route 22 rival caps at 8; Viridian Forest wild battles cap at 8; the Buggy 4 ramp from 7 to 9; Route 2 and the Pewter junior trainer cap at 10 before Brock's level 15 badge check. `rival-pallet-town` is an optional live-rival battle after `tutorial-16`, using up to three Pokemon from the selected rival's current Battle Team scaled to level 5. Winning or losing hides the battle and unlocks a short rival follow-up task.

The old extended tutorial beats now bridge the gap between the guided expedition and the broad path choices: `pallet-artisan-soft-touch-briefing` introduces the Soft Fluff/Artisan bench handoff, `pallet-artisan-basics` requires the one-time Quest recipe `craft-trainers-glove`, `tutorial-7` is a post-orientation Pokedex check after the glove patch, `tutorial-8` / `tutorial-9` introduce the Professor's TCG Maniac friend, and `tutorial-10` through `tutorial-15` run the Brock/Rhydon mix-up chain. The first TCG Maniac tutorial grants the Base Set binder, the photo follow-up grants a Base Set booster pack, and the Pallet TCG Maniac shop uses the dedicated `tcg-maniac-m` trainer sprite while selling only Base Set single booster packs plus 36-pack booster boxes priced at 30x the single-pack crystal cost until the player collects all 102 unique Base Set (`base1`) cards. Early bootstrap item bundles are intentionally small: the orientation field kit and `tutorial-6` each grant 5 Poke Balls, while Brock's research handoff grants 5 Battle Observers. The first `Rocky Cave` battle in that chain allows three player Pokemon at level cap 7, with zero-IV/zero-EV wild enemies.

- `explore-1` starts world exploration.
- `researcher-life-1` starts the research/stat scanner path.
- The Pallet stat module questline ends at `stat-module-3`, which requires Researcher level 7 before installing the `stats-scanner`; the Researcher skill guide lists this as "Stats Module".

Moon 1F/B1F/B2F, Route 4, Route 5, Route 24, Route 25, Route 6, Route 11, and Diglett's Cave. Shared desktop section emblems live under `public/ui/pokeori/desktop/` and are supplemental to accessible labels.
